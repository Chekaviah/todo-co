# Contributing

## Quick Guide

* [Fork](https://help.github.com/articles/fork-a-repo/) the repo.
* [Checkout](https://git-scm.com/docs/git-checkout) the `master` branch.
* Follow [README.md](https://github.com/Chekaviah/todo-co/blob/master/README.md) instructions to initialize the project.
* Create a new branch, e.g. `feature-foo` or `bugfix-bar`.
* Make changes
* If you are adding functionality or fixing a bug, implement functionals and units tests.
* Check if tests pass: `php bin/phpunit`.
* Commit and push changes.

## Opening a [Pull Request](https://help.github.com/articles/about-pull-requests/)

All merges must be approved by the lead developer. Pull requests ensure that functionals and units tests are successful and code quality is not degraded.

* Submit one Pull Request per fix or feature.
* If your changes are not up to date, [rebase](https://git-scm.com/docs/git-rebase) your branch onto the parent branch.##
